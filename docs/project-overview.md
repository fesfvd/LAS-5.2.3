LAS 文学分析系统项目概况

一、项目定位

LAS（Literary Analysis System）是一套基于大语言模型的十六维文学价值评估系统。用户提交作品后，系统以严肃文学史谱系为参照，对文本进行 16 个维度的逐项评分，并经体裁策略权重、偏科惩罚和平庸惩罚三重校准，最终输出包含雷达图、评分详表、深度文学分析和文学签文的交互式报告。

系统以 Web 应用形式交付（FastAPI 后端 + Vanilla JS SPA 前端），通过 SSE 流式传输实时展示分析进度。面向编辑审稿、写作教学、创作者自检三类场景。

二、当前进展

1. 理论框架：v5.2.3 版本已完成。包含 16 维评分锚点表、体裁自适应权重、L1-L5 基准搜索链、缺陷扫描、正缺陷豁免、原创审慎下调公式等完整机制。提示词采用 V2 JSON 模式（58KB 系统提示词，含 6 公理 + 13 步工作流 + 16 维锚定表 + 15 级标尺 + scoring_audit 自检），可通过环境变量切换至 V1 经典模式提示词。

2. 技术栈：DeepSeek V4 作为主推理引擎（1M 上下文），FastAPI + SQLAlchemy + SQLite 后端，Vanilla JS SPA 前端（hash 路由 + Chart.js 雷达图），SSE 流式传输。

3. 作品测评：计划对 565 部作品进行系统归档与测评，目前已完成 271 部。已测评作品涵盖东亚、欧洲、北美、拉美等 10 个区域，评分区间从《离骚》137.46 分到当代习作 32.35 分。

4. 外部合作：已获得本校樊老师初步反馈；与南开大学电子信息学院达成技术合作；启动资金 2000 元已自筹到位。

三、使用流程

系统提供两种分析模式：

1. 经典模式：仅需输入作品名（如《红楼梦》《百年孤独》）。系统利用 LLM 预训练语料中的全文知识进行分析，作者名由 LLM 自动补全。适合分析已出版的经典作品。

2. 原创模式：需填写标题、作者（选填）、粘贴正文。LLM 逐维度评分，所有 ≥75 分的维度须经过二次比较与降档调整（原创审慎校验），防止对未出版作品评分虚高。

分析过程按 5 个阶段实时流式传输进度（初始化 → 缺陷扫描 → 逐维评分 → 深度分析 → 综合计算），完成后自动跳转至报告页。

报告页提供：综合分环形动画、十六维雷达图、四层面水平柱状图、评分详表（支持档位筛选和关键词搜索，每行可展开查看基准比较、证据引用、反向锚定和结论五步）、核心基准集、深度文学分析手风琴、编辑审稿建议（原创模式）、创作导语（原创模式）、阅读建议、综合结论、附录（文脉/风物/余音/联结/延伸阅读）、文学签文。

四、核心方法简述

1. 16 个分析维度，分为四层面：
   A 语言与形式：语言艺术性、修辞运用、结构设计、文体适配
   B 叙事与内容：叙事技巧、人物塑造、情节架构、细节密度
   C 思想与意义：主题深度、情感力量、象征体系、时代价值
   D 审美与影响：风格独创性、审美统一、文学影响、经典地位

2. 评分机制：采用 0-150 分连续区间，15 档锚定标尺（从"文学之巅 👑 140-150"到"零价值 ∅ 0-4.9"）。每个维度须与《评分锚点表》中的守门员作品进行绝对比较，禁止在同类型圈层内做"相对优秀"的评判。每项得分须有至少两条具体文本证据支撑。

3. 体裁自适应：根据体裁匹配四种权重策略，例如分析诗歌时语言与形式层权重提升至 40%，分析小说时叙事与内容层权重提升至 35%。非叙事性体裁虽经权重调整，仍可能存在结构性偏差。

4. 基准选择：L1（同谱系同体裁）→ L2（同谱系跨体裁）→ L3（同文化传统）→ L4（跨文化同类型）→ L5（跨文化仅同层面）。核心基准集中至少 3 部来自 L1 或 L2，L4/L5 最多 1 部。

5. 五步法评分流程（每个维度独立执行）：
   第一步·查缺陷：若该维度被缺陷扫描命中，分数不得超过对应上限（≤55 或 ≤60）。
   第二步·反向锚定（原创模式）：与第九梯队"合格文本"比较，若未明显优于，分数强制 ≤54.9。
   第三步·正向对比：与正向基准比较，判定差距等级（显著/较大/明显/一定/接近/超越），产出具体比较表述。
   第四步·定档位：综合前三步结果，对照评分锚点表确定档位与分数（>60 分需至少 2 条证据）。
   第五步·给证据：提供至少 2 条具体文本引用，禁用"流畅""细腻"等空泛词。

6. 原创审慎校验：原始分数 ≥75 分的维度须进行二次比较。沿用原正向基准，根据偏差等级（微小/较小/中等/严重/极端）计算降档分数。数量超过 4 个时只取分最高的 4 个下调。

7. 正缺陷豁免：当某维度 ≥105 分且存在因果必然、不可规避、内在一贯的短板时，允许将该短板维度替换为其余 15 维度的平均值，不计入总分惩罚——体现公理 E（均衡性暴政抵抗）：偏科比平庸更接近文学本质。

8. 惩罚机制（仅原创模式）：
   - 偏科惩罚 k = 1/(1+0.01×gap)，核心维度薄弱时触发，k 不低于 0.65
   - 平庸惩罚 mf = 1-0.006×(75-mean)，前 14 维均分 < 65 且标准差 ≤ 8 时触发，mf 不低于 0.75
   WCS = PWS × k × mf（当惩罚未触发时 k=1, mf=1）

常见质疑应答速查

一、关于评分合法性
问：文学价值怎么能用分数衡量？这不是机械吗？
答：LAS 不取代批评判断，只提供可验证的参照。每个分数有三项基础：1）与该维度标杆作品的具体比较；2）至少两条原文证据；3）守门员作品的绝对锚点。类似于专业烹饪评审不说"好吃"，而说"脂肪比例偏高，建议延长煸炒 20 秒"。系统同时设有正缺陷豁免机制，允许天才偏科，不要求全面平庸。截至目前，LAS 已按此规则完成 271 部作品的测评（计划 565 部），覆盖从《离骚》137 分到当代习作 32 分的完整区间。

二、关于跨文化跨时代比较
问：不同文化、时代的作品怎么比较？用同一把尺子不公平。
答：系统采用 L1 至 L5 递进搜索链，优先使用同谱系、同体裁的基准。例如分析中国乡土小说，会先选《边城》《呼兰河传》，不直接套用西方作品。只有在本土谱系内找不到合适基准时，才会跨文化参照，且限制最多使用 1 部。另配体裁自适应权重，在不同体裁间调整维度权重（诗歌语言层权重 40%，小说叙事层权重 35%），以减少结构性偏差。

三、关于 AI 可信度与可复现性
问：AI 会不会乱评？换一个 AI 结果会不会不同？
答：所有评分规则（锚点表、缺陷扫描、审慎下调公式、豁免条件）均由人事先定义并固化在系统提示词中。AI 负责两件事：从文本中提取证据片段，以及判断该片段与基准作品片段的强弱关系。不同模型评分存在差异，同一模型对同一文本多次测评分数浮动在 5-7 分以内。系统不依赖单一分数，而是结合评级区间（如"典范之作"95-104.9 分）和决策日志共同呈现，每个维度的基准选择、证据引用和档位判定均可人工复核。当前主力模型为 DeepSeek V4。

四、与词频分析等工具的区别
问：这和市面上那些词频统计、情感分析有什么不同？
答：词频工具回答"这个文本有多少个成语、多少个形容词"。LAS 回答的是"这个作品在语言艺术性上与《边城》相比处于什么档位""在人物塑造上与《百年孤独》差几档"。它输出的是文学史坐标和具体的修改方向，而不是数据报表。

五、关于跨学科背景的质疑
问：你是会计专业，凭什么做文学分析工具？
答：正是因为跨学科，才能跳出传统文学批评的惯性依赖。方法论的核心是逻辑和可验证性，不依赖科班出身。这套框架的规则透明性正是源于工程思维对传统阐释型批评的重新组织。

六、关于应用场景与实用性
问：这个系统谁来用？有什么用？
答：第一服务对象是编辑和文学批评家，用于快速筛选稿件、定位作品优劣；第二服务对象是创作者，用于获得具体的修改建议（编辑审稿建议和创作导语模块直接输出诊断与改写方向）；第三服务对象是写作教学，辅助教师批改学生作业。系统提供的是参照而非判决，最终判断仍由人做出。目前已完成的 271 部测评覆盖东亚、欧洲、北美、拉美等 10 个区域，验证了框架在不同文化背景下的基本可用性。

七、关于失败或误判的可能
问：有没有可能系统把好作品打低分、把差作品打高分？
答：有可能。任何分析工具都有误判概率。LAS 通过以下方式降低风险：1）每个高分必须与高水准基准比较并给出证据；2）原创模式下 ≥75 分的维度会被二次降档校验；3）正缺陷豁免防止机械扣分；4）分析报告公开决策日志，允许用户质疑和复核。系统不宣称绝对正确，只提供可追溯的分析过程。

八、关于 AI 黑箱与可解释性
问：你说 AI 评的，但这个 AI 怎么评的，你自己也说不清楚吧？
答：评分规则是我预先定义的（16 个维度、锚点表、五步法），固化在系统提示词中。AI 做两件事：从文本里摘句子，以及判断这个句子"比基准好还是差"。它不需要解释"为什么好"，只需要做比较。每个判断都能追溯到原文位置。核心逻辑是人定的，AI 只是执行工具。决策日志中会列出每个维度用到的文本证据和比较结论，可供人工核验。

九、关于评分稳定性
问：同一篇文评两次，分数一样吗？
答：不一样。不同 AI 模型差别很大，同一个 AI 评两次也会有 5-7 分的浮动。所以分数不当作绝对精确值，而是看它落在哪个评级区间（比如"典范之作"95-104.9 分）。评级比具体分数更可靠。系统报告中同时给出分数和评级，并建议用户重点关注评级和分析而非小数点后的精确值。

十、关于维度重叠
问："语言艺术性"和"修辞运用"有什么区别？感觉是同一回事。
答：确实高度相关。修辞运用更像语言艺术性的一个子集。目前的做法是两者都保留，但权重都不高（各 6% 左右），加权后对总分影响有限。在已测评的 271 部作品上做过相关性观察，这两个维度的分数走向基本一致，但尚未做严格的因子分析。长期考虑合并或进一步验证。

十一、关于"审美统一"这种主观维度
问："审美统一"这种判断，AI 怎么评？
答：这是最难的维度之一。AI 不是直接判断美不美，而是检测具体特征：风格有没有突变、意象有没有断裂、语调是否一致。目前这个维度权重很低（5%）。

十二、关于权重比例的依据
问：A 层 25%、B 层 35%，这个比例有什么依据？
答：目前是项目负责人的初始设定，不是实证结论。计划在测评更多作品后，通过问卷征求文学老师们的意见来修正。

十三、关于经典固化与文化偏差
问：你基准库里全是经典，那网络小说是不是天生就低分？
答：系统使用的基准信息来自大模型的预训练语料，经典作品在语料中占比高，网络小说等新作品占比低，这确实可能导致分析时偏向经典标准。这是 LLM 作为底层引擎的固有约束，V2 提示词中已加入体裁自检机制来部分缓解此偏差。

十四、关于下限参照 N/A 的问题
问：报告里下限参照全是 N/A，那你怎么知道 50 分是什么水平？
答：经典模式（分析已出版的经典作品）不需要下限参照，因为锚点表里每个档位都有守门员作品。原创模式下每个维度必须指定一个"合格文本"作为下限，比如用《斗罗大陆》的语言水平作为 50 分的参照。如果看到 N/A，是因为该作品走的是经典模式。如需查看原创模式的完整示例，可单独提供原创作品的分析报告。

十五、关于内部知识库与模型能力
问：你比较时用的《边城》《围城》《百年孤独》这些基准作品，系统是怎么知道的？
答：这些信息来自大模型预训练时内置的知识库（即预训练语料）。不同模型预训练的语料规模不同，越好的模型知道的经典作品越多、对文本的理解能力也越强。LAS 本身不维护作品库，而是依赖底层大模型已经学到的文学知识来提取基准特征和执行比较。因此，系统的分析质量直接受所选大模型的影响。当前推荐的模型是 DeepSeek V4 Pro（1M 上下文窗口，适合长文本分析）。
