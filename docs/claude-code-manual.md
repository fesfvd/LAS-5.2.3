## 1. 基础概念

### Claude Code 是什么

Claude Code 是 Anthropic 的 AI 编程助手，可以读写你的项目文件、执行命令、搜索代码库。它运行在 VS Code 扩展中（也有独立终端版），你在聊天面板里用自然语言描述需求，它直接改代码。

### 它能做什么

- 读写和编辑项目文件
- 执行终端命令（`git`、`npm`、`pip`、`python` 等）
- 搜索代码库（按文件名、按内容、按正则）
- 启动和管理子任务（并行处理多个独立工作）
- 记住项目上下文和你的偏好

### 它不能做什么

- 访问外部网络（除非你授予 WebFetch/WebSearch 权限）
- 主动推送代码到远程仓库
- 在你的机器上安装系统级软件
- 替换你的判断——关键决策始终由你确认

## 2. VS Code 扩展界面

### 打开方式

- 点击编辑器右上角的 **星形图标**（Claude Code 图标）
- `Ctrl+Shift+P` → 搜索 "Claude Code: Open in New Tab"
- 点击 VS Code 底部状态栏的 `Claude Code`

### 界面布局

```
┌─────────────────────────────────────────────┐
│  [标签页1] [标签页2] [+]        ···          │  ← 多会话标签
├─────────────────────────────────────────────┤
│                                             │
│  对话内容区                                   │
│  · Claude 的文字回复                          │
│  · 代码 diff（绿色=新增，红色=删除）            │
│  · 工具调用记录                               │
│                                             │
├─────────────────────────────────────────────┤
│  [输入框]                          [模式指示] │  ← 底部输入区
└─────────────────────────────────────────────┘
```

**标签页**：每个标签是独立会话。`Ctrl+N` 新建，`Ctrl+W` 关闭。不同任务用不同标签，不要把修 bug 和加功能混在一个对话。

**内联 Diff**：Claude 编辑文件后，VS Code 显示左右对照的差异视图。每处修改有 **Accept** 和 **Reject** 按钮，你可以接受部分改动、拒绝另外部分——这是 VS Code 扩展版相比终端版的核心优势。

**模式指示器**：右下角显示当前权限模式（Edit / Auto-Accept / Plan），点击可切换。

### 状态栏

底部状态栏显示：
- 当前模型（如 `opus`、`sonnet`）
- 权限模式
- 会话状态

## 3. 斜杠命令速查

在输入框输入 `/` 弹出命令菜单。

### 会话管理

| 命令 | 作用 | 使用场景 |
|------|------|---------|
| `/clear` | 清空整个对话 | 上下文太臃肿，需要重新开始 |
| `/compact` | 压缩对话（保留关键摘要） | **优先用这个**——释放 token 但不丢上下文 |
| `/rewind` | 撤销上一条交互（代码+对话） | Claude 改错了，回退 |
| `/diff` | 查看当前会话所有文件改动 | 提交前审核 |
| `/copy` | 复制最后一条回复到剪贴板 | 需要粘贴到别处 |

### 模式切换

| 命令 | 作用 |
|------|------|
| `/model opus` | 切换到最强模型（复杂推理） |
| `/model sonnet` | 切换到快速模型（日常编码） |
| `/effort high` | 提高推理深度（硬 bug 调试） |
| `/fast` | 切换快速输出模式 |

> 你的环境通过代理路由到 DeepSeek，实际模型取决于配置。用 `/status` 查看当前模型。

### 项目管理

| 命令 | 作用 |
|------|------|
| `/init` | 生成/更新 `CLAUDE.md` 项目文档 |
| `/plan` | 进入只读规划模式（先分析后动手） |
| `/review` | 审查当前分支的待提交改动 |
| `/security-review` | 安全审计 |
| `/simplify` | 检测过度工程化 |

### 诊断

| 命令 | 作用 |
|------|------|
| `/doctor` | 环境诊断（出问题时先跑这个） |
| `/usage` | 查看 token 消耗和费用 |
| `/context` | 上下文占用可视化 |
| `/status` | 当前模型、版本、账户 |
| `/insights` | 生成会话历史 HTML 报告 |

## 5. 三种权限模式

在输入框旁或 `Shift+Tab` 循环切换。

### Edit（默认）

Claude 提出修改方案，你逐项审核 diff，选择接受或拒绝。

**什么时候用**：正常的日常开发。每个文件改动都会经过你。

### Auto-Accept

Claude 直接写入文件，不弹 diff 确认。

**什么时候用**：脚手架搭建、批量重命名、模板代码生成等可信任的机械操作。

> Auto-Accept 模式下仍可通过双击 `Esc` 回退到上一个检查点。

### Plan

Claude 只能读取和搜索代码，不能修改任何文件。

**什么时候用**：探索未知代码逻辑、架构评估、代码审查、影响分析。在动手前先摸底。

## 6. 引用文件与上下文管理

### @-mention 文件引用

输入框中输入 `@`，弹出文件名模糊搜索。选择文件后，Claude 会读取该文件内容作为上下文。

```
# 示例
帮我分析 @backend/main.py 中的路由结构
修复 @frontend/js/pages/upload.js:85-95 的 bug
```

你也可以拖动文件到输入框直接引用。

### 上下文生命周期

- 每个对话标签有独立的上下文窗口
- 当对话过长，系统会自动**压缩**早期消息（保留摘要）
- 压缩后 Claude 仍知道之前做了什么，但细节可能丢失

### 什么时候该 `/compact`

- 对话超过 40-50 轮
- Claude 开始"忘记"你之前告诉它的信息
- 回复质量明显下降

### 什么时候该 `/clear`

- 切换到一个完全不相关的任务
- 上下文被错误的方向污染（反复在一个思路上打转）
- 想完全重新开始

> `/compact` 保留关键信息 + 释放 token，比 `/clear` 更温和。除非确实需要推倒重来，优先 `/compact`。

### 多标签管理

```
标签1: 功能开发（长期会话）
标签2: bug 修复（快速会话）
标签3: 代码审查（临时会话）
```

不要在一个标签里混多个独立任务。上下文是宝贵资源，混任务会互相污染。

## 7. LAS 自定义技能

在输入框输入 `/` 可看到所有已安装技能。以下是 LAS 项目的 5 个专属技能：

### `/las-debug` — 全链路诊断

**用途**：报告空白、数据缺失、分析失败、脚本报错时使用。

**原理**：自动追踪 LLM 输出 → 后端 JSON 解析 → API 响应 → 前端渲染，逐层定位断点。

**使用**：
```
/las-debug
分析完成后报告页一片空白
```

不需要告诉它哪里出了问题——技能会自动排查整个链路。

### `/las-smoke` — 端到端冒烟测试

**用途**：改完代码后快速验证全链路通畅。部署前检查。

**原理**：用极短文本秒级完成一次分析，验证提交→分析→报告的完整流程。

**使用**：
```
/las-smoke
```

每次改完代码后跑一次，防止引入回归。

### `/code-reviewer` — 前端代码审查

**用途**：提交前端代码前做多维度审查。

**审查维度**：语义化 HTML、可访问性（a11y）、性能（渲染、加载）、安全性（XSS、注入）、可维护性（代码组织）。

**使用**：
```
/code-reviewer
审查最近改动的 report.js
```

### `/design-to-code` — 设计还原

**用途**：将 Figma 设计稿、截图或自然语言描述的 UI 需求转化为前端代码。

**使用**：
```
/design-to-code
在提交页添加一个模型选择下拉框，参考原有 mode-switch 风格
```

### `/frontend-designer` — 前端设计美学

**用途**：提升 AI 生成 UI 的审美水平与工程质量，确保输出符合 LAS 设计系统。

**使用**：
```
/frontend-designer
帮我设计一个分析历史的列表卡片
```

### 其他通用技能

| 技能 | 用途 |
|------|------|
| `/debug` | 通用 bug 调试 |
| `/write-plan` | 多步实现前的结构化规划 |
| `/specify` | 将自然语言需求转为正式规格说明 |
| `/clarify` | 识别需求中的模糊点并提问澄清 |
| `/verify` | 验证修复/功能是否真的完成 |
| `/perf-profile` | 性能瓶颈分析 |
| `/tdd` | 测试驱动开发（先写测试再写实现） |
| `/security-review` | 安全审查 |
| `/simplify` | 过度工程化检测 |

## 8. LAS 项目开发流程

### 日常开发闭环

```
提出需求 → /write-plan（规划）→ 实现 → /las-smoke（冒烟）→ /verify（确认）→ 提交
```

### 修复 bug

```
描述现象 → /las-debug（诊断）→ 定位根因 → 修复 → /las-smoke → /verify → 提交
```

### 新增前端功能

```
描述需求 → /frontend-designer（设计）→ /design-to-code（还原）→ /code-reviewer（审查）→ 提交
```

### 安全审查（定期）

```
/security-review
```

审查当前分支所有改动中的安全隐患（XSS、注入、密钥泄露等）。

### 提交前检查清单

| 步骤 | 命令 |
|------|------|
| 1. 跑冒烟测试 | `/las-smoke` |
| 2. 审核所有改动 | `/diff` |
| 3. 前端代码质量 | `/code-reviewer` |
| 4. 安全检查 | `/security-review` |
| 5. 生成 commit 信息 | `帮我根据当前改动写一个 commit message` |

### 使用 Git 工作树隔离

当你想在隔离环境中实验而不影响主工作区：

```
/ 帮我创建一个 worktree 来测试新的评分公式
```

Claude 会在 `.claude/worktrees/` 下创建独立的 git 工作树，实验完成后可以保留或删除。

## 9. 配置与设置

### 配置文件位置

| 文件 | 路径 | 作用 | 是否提交 Git |
|------|------|------|-------------|
| 项目设置 | `d:\LAS 5.2.3\.claude\settings.json` | 项目级配置 | 是 |
| 个人设置 | `d:\LAS 5.2.3\.claude\settings.local.json` | 个人密钥/偏好 | 否 |
| 用户全局 | `~/.claude/settings.json` | 所有项目共享 | 否 |
| 项目文档 | `d:\LAS 5.2.3\CLAUDE.md` | 项目说明（AI 自动读取） | 是 |
| 个人文档 | `d:\LAS 5.2.3\CLAUDE.local.md` | 个人补充说明 | 否 |

### 权限配置

在 `settings.json` 中配置工具权限：

```json
{
  "permissions": {
    "allow": [
      "Bash(npm install *)",
      "Bash(pip install *)",
      "Bash(git: *)"
    ],
    "deny": [
      "Bash(rm -rf / *)",
      "Read(.env)"
    ]
  }
}
```

- `allow`：自动允许，不弹确认框
- `deny`：始终禁止
- 未列出的操作会弹框询问

### 减少权限弹窗

如果某些只读命令反复弹窗，可以运行：

```
/fewer-permission-prompts
```

它会扫描你的使用历史，自动建议一个优化后的 allowlist 写入 `settings.json`。

### 自定义斜杠命令

在 `.claude/commands/` 下创建 `.md` 文件即可定义新命令。

**示例** — 创建 `d:\LAS 5.2.3\.claude\commands\restart.md`：

```markdown
重启 LAS 开发服务器：

```bash
python -m uvicorn backend.main:app --host 127.0.0.1 --port 8000 --reload
```
```

之后输入 `/restart` 即执行。

### 环境变量

在 `settings.json` 中设置，无需修改系统环境：

```json
{
  "env": {
    "PYTHONPATH": "d:/LAS 5.2.3",
    "DEBUG": "1"
  }
}
```

### MEMORY.md 自动记忆

Claude Code 会在 `~/.claude/projects/d--LAS-5-2-3/memory/` 下维护持久记忆，跨会话保留：

- **user** — 你的角色、偏好、知识背景
- **feedback** — 你对工作方式的反馈（"以后都这样做"/"别再这样了"）
- **project** — 项目目标、约束、截止日期
- **reference** — 外部资源引用（文档链接、看板地址等）

你可以直接说"记住这个"或"以后每次改完代码都跑 /las-smoke"，Claude 会自动写入记忆。

## 10. 常见问题

### Claude 改错了，怎么回退？

双击 `Esc` 回退到上一个检查点。如果想回退更远，可以多次双击。

### 上下文满了怎么办？

1. 先用 `/compact`（压缩保留摘要）
2. 如果还不够，`/clear` + 重新描述当前任务
3. 把大任务拆成多个小任务，用不同标签

### 怎么让 Claude 更快？

- `/model haiku` 切换到更快模型（适用于简单任务）
- `/fast` 开启快速输出模式
- 单次输入描述清楚需求，减少来回确认

### 怎么让 Claude 回答更准确？

- 用 `@` 精确引用相关文件
- 描述需求时给出具体上下文（"修复上传按钮在 480px 宽度下错位" 比 "修 bug" 好）
- 复杂的改动先 `/write-plan` 规划再执行

### 我能信任 Auto-Accept 模式吗？

可以，但有前提：
- 你清楚知道要做什么（不是探索性任务）
- 改动范围明确（单个文件的批量替换、模板生成）
- 改完后用 `/diff` 审核

不确定时保持 Edit 模式，逐项审核 diff。

### 怎么知道花了多少钱？

`/usage` 查看当前会话的 token 消耗。你的环境路由到 DeepSeek，费用按 DeepSeek 定价计算。
