# Frontend Designer — 前端设计审美与工程规范 SKILL

当你被要求编写前端代码、设计 UI 组件、或评审设计时，请严格遵循以下准则。目标是产出兼具视觉美感、交互愉悦、工程健壮与包容性的现代前端界面。设计不仅是装饰，更是功能的延伸与对细节的极致把控。

---

## 🌟 核心设计哲学

- **少即是多，但绝非空洞**：去除一切不必要的装饰，让每一个像素都有存在的意义。
- **内容优先，形式追随**：设计是为了让内容更好地被消费，而不是阻碍内容。
- **一致性即美感**：建立并严格遵循设计系统，混乱是审美的天敌。
- **有灵魂的细节**：微交互、恰到好处的阴影、精心调配的色彩，是区分"平庸"与"卓越"的关键。
- **美学与无障碍并重**：没有可及性的美感是残缺的，真正的现代设计必须包容所有用户。

---

## 🎨 一、视觉与排版

色彩是界面的情绪，排版是界面的声音，留白是最高级的奢华。

### 1.1 色彩系统
- **克制用色（60-30-10）**：60% 主色/背景，30% 辅色/组件，10% 强调色/行动点。不要把界面变成调色盘。
- **色阶体系**：永远不要只用一个纯色。定义从 50 到 950 的色阶系统，浅色用于背景，深色用于文本。
- **现代阴影**：摒弃生硬纯黑投影。使用基于主色的低饱和度半透明阴影（如 `rgba(0,0,0,0.08)`），多层叠加比单层重影更通透。
- **深色模式**：深灰底（如 `#121212`）配浅色文字，注意对比度降级，避免刺眼纯白大段文本。通过 `data-theme="dark"` 或 `prefers-color-scheme` 实现主题切换，语义 Token 自动映射。

### 1.2 排版体系
- **8点网格系统**：所有间距、尺寸优先使用 4 或 8 的倍数（4, 8, 12, 16, 24, 32, 48, 64）。消灭随意的 13px 或 17px。
- **亲密性与呼吸感**：间距阶梯（8px 元素内距, 16px 组件内距, 32px 模块间距）传达层级；内容绝不紧贴屏幕边缘。
- **对齐原则**：视觉对齐优先于数值对齐，允许子像素微调（如 `transform: translateX(-50%)`）。全局保持逻辑一致。
- **字号阶梯**：建立严格 Typography Scale（12, 14, 16, 20, 24, 32, 48），字重对比（400 vs 600/700）要足够明显。
- **行高与行长**：正文行高 1.5-1.8，标题 1.2-1.4。单行中文 35-50 字，英文 50-75 字。

### 1.3 图标系统
- **SVG 优先**：使用 SVG 图标，优于图标字体（避免渲染模糊与可访问性问题）。
- **尺寸阶梯**：图标尺寸遵循 16/20/24px 阶梯，与文本基线对齐。
- **可访问性**：图标必须携带 `aria-label` 或与文本配对。禁止单独出现的功能性图标无替代文本。

## 🧩 二、交互与状态

组件是积木，交互是灵魂，状态流转决定了体验的平滑度。

### 2.1 状态完整性
任何可交互元素必须提供视觉反馈：
- **Default** → **Hover** → **Active/Focus** → **Disabled** → **Loading** → **Selected**
- **Error**：操作失败、网络中断、校验不通过时的明确视觉提示（红色边框/图标/文案，禁用恐慌红大面积铺色）。
- **Empty**：无数据、无结果、未初始化时的占位状态（插图 + 引导文案 + 操作按钮，禁止裸奔白屏）。
- **Skeleton**：数据加载时，使用与最终结构一致的灰色区块占位，避免布局偏移（CLS）。

没有反馈的按钮是死物。

### 2.2 动效与反馈
- **圆角的性格**：大圆角（12px+）亲和现代；小圆角（4-8px）严谨专业；直角（0px）硬朗前卫。全局必须统一。
- **动效克制**：引导视线或提供反馈，时长 200ms-500ms。使用缓动函数（`ease-out` 或 `cubic-bezier`），拒绝生硬线性运动。
- **乐观更新**：用户提交操作（如点赞、收藏）时，UI 立即切换至成功状态，后台异步处理，换取极致流畅感。
- **边界处理**：内容超出时优雅降级（截断、省略号、滚动条），任何溢出必须有降级策略，避免布局崩塌。

### 2.3 表单与输入
- **标签对齐**：顶对齐优先于左对齐，缩短眼动路径；单列布局优于多列。
- **触摸区域**：输入框与按钮的最小触摸区域不低于 44×44px（WCAG 2.5.5）。
- **校验策略**：即时校验用于格式错误（如邮箱格式），提交后校验用于业务逻辑（如用户名重复）。错误提示紧贴输入框，使用图标 + 红色文案。
- **自动填充**：覆盖浏览器默认的 `autofill` 背景色（如 `box-shadow: 0 0 0 1000px white inset`），保持视觉一致。
- **焦点管理**：表单提交后焦点应转移至第一个错误字段或成功提示。

## 📐 三、响应式与跨端

界面不是静态画布，而是流动的容器。

- **流式排版**：摒弃断点生硬跳变，使用 `clamp()` 实现字号/间距平滑缩放（如 `font-size: clamp(16px, 2vw, 24px)`）。
- **容器查询**：组件响应式应基于父容器宽度而非视口，使用 `@container` 让组件具备自适应智能。
- **逻辑属性**：全球化产品必须使用 `margin-inline-start` 替代 `margin-left`，原生支持 RTL 布局。
- **触摸优化**：移动端消除 300ms 点击延迟（`touch-action: manipulation`）；禁止文本误选需审慎使用（`-webkit-user-select` 仅在交互组件上限制，正文保持可选）。

## ♿️ 四、无障碍与包容性

真正的现代设计必须包容所有用户，这是强约束而非可选项。

- **色彩对比度**：正文与背景对比度必须达到 WCAG AA 标准（至少 4.5:1），浅灰文字（如 `#999`）慎用。
- **动效减弱偏好**：使用 `@media (prefers-reduced-motion: reduce)` 为前庭功能障碍用户关闭非必要动画。
- **焦点可见性**：永远不要全局 `outline: none`。使用 `:focus-visible` 仅为键盘导航用户提供清晰焦点环。
- **语义化优先**：用原生 HTML 元素（`button`, `a`, `label`, `input`）及 ARIA 属性构建交互，拒绝纯 `div`/`span` 堆砌。
- **键盘导航**：所有交互组件必须支持 Tab 顺序、Enter/Space 触发、Esc 关闭。

## 💻 五、工程与架构

好的设计需要好的代码承载，代码本身也是一种美学。

### 5.1 Design Tokens 分层
CSS 变量分为：
- **全局语义层**：`--color-primary`, `--color-text-secondary`
- **组件应用层**：`--button-bg: var(--color-primary)`, `--input-border: var(--color-border)`
主题切换只改变语义层，组件层自动继承。

### 5.2 现代布局方案
- 放弃繁琐嵌套，使用 `grid-template-areas` 或 `subgrid` 解耦 DOM 与视觉布局。
- 最大嵌套深度不超过 3 层，保持 DOM 扁平。

### 5.3 CSS 架构
- **层级管理**：建立全局 Z-Index 变量（如 `--z-dropdown: 100`, `--z-modal: 200`, `--z-toast: 300`），禁止随意写死 `z-index: 9999`。
- **方法论**：默认采用 Utility-first（如 Tailwind）或 BEM，保持命名空间清晰；组件样式优先隔离（CSS Modules / Scoped CSS）。
- **禁止行内硬编码**：所有颜色、间距、阴影必须抽取为 CSS 变量，遵循 Token 分层原则。

### 5.4 性能策略
- **渐进式揭示**：复杂图表/长列表采用懒加载与虚拟滚动，确保首屏渲染极速。
- **图片策略**：使用 `loading="lazy"` + 模糊占位图（LQIP），关键图片预加载。
- **字体加载**：使用 `font-display: swap` 避免 FOIT（Flash of Invisible Text），提供系统字体回退栈。
- **关键路径**：关键 CSS 优先内联，非关键资源异步加载。

## 🚀 六、扩展范式（按需启用）

以下规范仅在用户明确要求 AI 交互组件、沉浸式界面或未来设备适配时启用，常规 UI 默认不触发。

- **AIGC 交互组件**：流式输出打字机效果（光标闪烁频率 530ms，逐字延迟 30-80ms）、可信度指示器（置信度色标）、重新生成/停止控制（固定悬浮于输出区域右下角）。
- **空间计算暗示**：通过玻璃拟态（`backdrop-filter: blur(12px)`）和景深模糊，在扁平界面中暗示 Z 轴层级。常规界面避免滥用，防止可读性下降。

## ⚠️ 强制输出规范

当生成前端代码时，必须遵守以下底线：

- **文件组织**：默认输出可运行的单文件代码（HTML + CSS + JS 合一）。若涉及多组件协同、复杂状态管理或用户明确要求工程化结构，则输出模块化拆分方案（HTML/CSS/JS 分离）。
- **状态样式**：必须包含基础的状态样式（至少包含 Hover、Focus/Active、Disabled 状态）。
- **Token 化**：禁止使用行内硬编码颜色/间距，必须抽取为 CSS 变量（遵循 Token 分层原则）。
- **可访问性**：交互组件必须包含 ARIA 属性，并处理键盘 Enter/Space 触发事件。
- **响应式**：布局必须具备基础响应式能力，不能在窄屏上破版或溢出。
- **空状态与错误**：任何数据容器必须包含 Empty 与 Error 状态的预设占位，禁止裸奔白屏。
